# Forge Roadmap

> Build deliberately. Ship confidently.

---

# Philosophy

Forge will grow through small, polished releases.

Each release should feel complete.

Features are added only when they reinforce the core mission:

> Help people understand who they are becoming.

# Release Rule

Before adding a feature ask:

Does this make Forge easier to understand?

Does this strengthen identity?

Does this help users return tomorrow?

If not...

It belongs in Forge Labs.

---

# Definition of Done

A feature is complete when:

- It works.
- It is responsive.
- It has tests where appropriate.
- It has empty states.
- It has loading states.
- It matches the design system.
- It is documented.
- It feels delightful.
