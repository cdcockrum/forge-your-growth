# Forge Design System

> Calm. Intentional. Intelligent.

Forge should never feel like enterprise software.

It should feel like sitting at a beautiful wooden desk each morning to reflect, plan, and grow.

---

# Design Philosophy

Forge is not a dashboard.

Forge is an observatory.

Users are not managing tasks.

They are observing the evolution of themselves.

Every design decision should reinforce this feeling.

---

# Emotional Goals

Users should feel:

• Calm

• Curious

• Capable

• Intentional

Never:

• Guilty

• Overwhelmed

• Rushed

• Judged

---

# Visual Language

Imagine a cross between:

• Apple Health

• Arc Browser

• Notion

• Readwise

• Oura

• Obsidian

Minimal.

Thoughtful.

Warm.

Elegant.

# Typography

Display

Bold

Large

Used only for page titles.

Heading

Semibold

Clear hierarchy.

Body

Comfortable reading.

Captions

Muted.

Never tiny.

---

# Corners

Cards

rounded-2xl

Buttons

rounded-xl

Badges

rounded-full

Maintain consistency.

---

# Shadows

Very subtle.

Cards should feel elevated,

not floating.

---

# Motion

Animations should communicate state.

Never decoration.

Examples

Identity Tree grows.

Progress bars fill.

Charts animate.

Cards fade.

Nothing bounces.

---

# Spacing

Generous whitespace.

Content should breathe.

Prefer removing elements over shrinking them.

---

# Icons

Lucide only.

Consistent stroke width.

Avoid mixing icon libraries.

---

# Components

Everything should be reusable.

ForgeCard

ForgeMetric

ForgeChart

ForgeBadge

ForgeButton

ForgeSection

ForgeTimeline

ForgeProgress

ForgeEmptyState

ForgeSkeleton

---

# Cards

Every card answers one question.

Never multiple.

Examples

"What happened?"

"What should I do?"

"Who am I becoming?"

---

# Charts

One hero chart.

Supporting charts secondary.

Charts explain.

They do not decorate.

---

# Identity

Identity is Forge's signature.

Every interaction should reinforce it.

Practice does not increase streaks.

Practice adds evidence.

Evidence strengthens identity.

Identity changes over time.

---

# Empty States

Never blank.

Always educational.

Example

Forge is still gathering evidence.

Complete a few more sessions to unlock pattern discovery.

---

# Loading

Use skeletons.

Never spinning loaders unless unavoidable.

---

# Mobile Philosophy

Do not shrink desktop.

Redesign for touch.

Prioritize

Morning Briefing

↓

Today's Practice

↓

Identity

↓

Insights

↓

History

---

# Accessibility

Keyboard first.

High contrast.

Large tap targets.

Clear focus states.

Animations respect reduced motion.

---

# Performance

Fast feels intelligent.

Prefer instant feedback.

Lazy-load heavy visualizations.

Avoid unnecessary re-renders.

---

# Final Rule

Every screen should answer one question.

If it answers two,

split it.

If it answers none,

remove it.
